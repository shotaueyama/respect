# RESPECT FORCE WordPress 操作マニュアル

最終更新: 2026-07-07

このマニュアルは、RESPECT FORCE 新サイトの WordPress 管理画面で日常的に行う操作をまとめたものです。管理者ID、パスワード、サーバー接続情報は記載しません。

## 1. 管理画面へのログイン

管理画面URL:

```text
https://test.respect-force.co.jp/wp-admin/
```

本番移管後はドメインのみ本番に置き換わります。

```text
https://respect-force.co.jp/wp-admin/
```

ログイン後、左メニューから固定ページ、投稿、カテゴリー、設定などを操作します。

## 2. ページ構成の考え方

## 3. 固定ページを編集する

操作手順:

1. 管理画面にログインします。
2. 左メニューの「固定ページ」を開きます。
3. 編集したいページを選択します。
4. タイトル、本文、テンプレート、カスタムフィールドを確認します。
5. 変更後、「更新」を押します。

注意点:

- テンプレートを変更するとページの見た目や機能が大きく変わるため、理由がない限り変更しないでください。
- `buying.html`、`relabs.html`、`share-salon.html` などは専用テンプレートで大部分がコード管理されています。本文を編集してもページ上に反映されない箇所があります。
- トップページは「トップページ」テンプレートを割り当てた固定ページとして登録できます。

## 4. トップページの注目機器ランキングを並び替える

トップページの「注目機器ランキング」は、固定ページ編集画面からドラッグ&ドロップで並び替えできます。

操作手順:

1. 管理画面の「固定ページ」を開きます。
2. トップページとして使っている固定ページを開きます。
3. 右側または下部にある「注目機器ランキング 並び順」を探します。
4. 項目左側の「≡」をドラッグして順番を変更します。
5. 並び替え後、自動保存メッセージが表示されることを確認します。
6. 念のため固定ページ自体も「更新」します。
7. トップページを開き、表示順が変わっているか確認します。

注意点:

- この並び替え欄は「トップページ」テンプレートが設定された固定ページでのみ有効です。
- 並び順が保存されない場合は、ブラウザキャッシュを削除するか、別ブラウザで管理画面を開いて確認してください。

## 5. 商品を登録・編集する

商品は WordPress の「投稿」として管理されています。商品カテゴリー配下に登録され、商品詳細ページでは `detail_id` を使って表示されます。

操作手順:

1. 管理画面の「投稿」を開きます。
2. 新規追加または既存投稿を編集します。
3. タイトルに商品名を入力します。
4. 商品に合うカテゴリーを選択します。
5. 必要なカスタムフィールドを入力します。
6. アイキャッチ画像を設定します。
7. 「公開」または「更新」を押します。

主なカスタムフィールド:

| フィールド | 用途 |
| --- | --- |
| `connect_id` | 旧商品DBまたは商品詳細データと紐づけるID |
| `ranking` | 商品一覧の「人気順」で使用 |
| `attention` | 商品一覧の「注目順」で使用 |
| `recommendation` | おすすめ度として管理 |
| `topic_id` | 旧ニュースIDとの紐づけに使用 |
| `custom_permalink` | 旧URLを維持するためのURL指定 |

注意点:

- 商品詳細URLは原則 `/detail.html?detail_id=商品ID` の形式を維持します。
- 売約済み商品は `/detail_outlet.html?detail_id=...` として旧URLを維持しています。
- `connect_id` や旧IDを誤ると、商品詳細の内容が別の商品になる可能性があります。
- カテゴリーを外すと商品一覧に出ないことがあります。

## 6. 商品のランキング値をクイック編集する

投稿一覧では、`ranking`、`attention`、`recommendation` をクイック編集できます。

操作手順:

1. 管理画面の「投稿」を開きます。
2. 対象商品の「クイック編集」を押します。
3. 「ランキング」「注目度」「おすすめ度」を入力します。
4. 「更新」を押します。

注意点:

- 空欄または `0` を入れると、その値は削除されます。
- 商品一覧の並び替えで使うため、数値の意味を運用ルールとして揃えてください。

## 7. 商品カテゴリーを管理する

商品一覧のカテゴリーページは WordPress のカテゴリーで管理されています。

操作手順:

1. 管理画面の「投稿 > カテゴリー」を開きます。
2. 商品カテゴリー配下に必要なカテゴリーを作成します。
3. カテゴリー名、スラッグ、親カテゴリーを設定します。
4. 旧URLとの対応が必要な場合は、カスタムフィールド `list_id` を確認します。

注意点:

- 旧サイトの `list.html?list_id=...` は `list_id` を使って新サイトの商品カテゴリーに紐づきます。
- カテゴリー名を変更すると表示名が変わります。
- スラッグを変更すると新URLが変わるため、SEO移管中は慎重に扱ってください。

## 8. 商品一覧の絞り込み・並び替え

商品一覧ページでは以下の並び替えが使われます。

| 表示名 | 参照する値 |
| --- | --- |
| 人気順 | `ranking` |
| 注目順 | `attention` |
| 新着順 | 投稿日 |
| 価格が安い順 | 商品価格 |
| 価格が高い順 | 商品価格 |

価格や表示情報は商品詳細データから取得されます。商品カードに価格が出ない場合は、商品側の価格データまたは紐づけIDを確認してください。

## 9. ニュース記事を登録・編集する

ニュースは「投稿」として管理されています。

操作手順:

1. 管理画面の「投稿」を開きます。
2. 新規追加または既存投稿を編集します。
3. タイトル、本文、カテゴリーを入力します。
4. 必要に応じてアイキャッチ画像を設定します。
5. 「公開」または「更新」を押します。

旧URL維持に関する重要項目:

| フィールド | 用途 |
| --- | --- |
| `topic_id` | 旧ニュースURLの数字ID |
| `custom_permalink` | 例: `news1863.html` |

注意点:

- 旧URL `news1863.html` の数字は、WordPress投稿IDとは限りません。
- 正しい記事を表示するには、`topic_id=1863` または `custom_permalink=news1863.html` が必要です。
- 旧URLの記事内容が違う場合は、まず `topic_id` と `custom_permalink` を確認してください。

## 10. お問い合わせフォームを管理する

通常のお問い合わせページは固定ページと Contact Form 7 のフォームを利用しています。

操作手順:

1. 管理画面の「固定ページ」を開きます。
2. `contact.html` に対応するお問い合わせページを確認します。
3. フォーム本体を変更する場合は「お問い合わせ」メニュー、または Contact Form 7 の管理画面を確認します。

注意点:

- 固定ページテンプレート側でフォームの見た目を整形しているため、フォーム項目名を変更すると表示や送信処理に影響することがあります。
- メール送信先を変更する場合は、フォーム側のメール設定も確認してください。

## 11. 商品詳細ページのお問い合わせメールを管理する

商品詳細ページの「この商品について問い合わせする」は、専用の独自送信処理を使っています。

設定場所:

```text
管理画面 > 設定 > 商品問い合わせメール
```

設定できる項目:

| 項目 | 内容 |
| --- | --- |
| 送信先メールアドレス | 商品問い合わせの送信先 |
| 本文テンプレート | 管理者宛メールの本文 |

利用可能タグ:

```text
{product_title}
{product_url}
{inquiry_type}
{your_name}
{company_name}
{tel}
{your_email}
{message}
```

注意点:

- 送信先メールアドレスを空欄にすると、デフォルトの管理者メールが使われます。
- 商品名を本文に自動挿入するには `{product_title}` を本文テンプレートに含めてください。
- 現在の運用では送信先は `order@respect-force.co.jp` を基本にしてください。

## 12. SEOタイトル・ディスクリプションを確認する

主要ページにはテンプレート側で `title` と `meta description` が設定されています。

確認方法:

1. 対象ページをブラウザで開きます。
2. ページのソースを表示します。
3. `<title>` と `<meta name="description">` を確認します。

主なSEO設定はテーマの `functions.php` 内に定義されています。

注意点:

- 商品詳細ページは商品名と商品説明から自動生成されます。
- ニュース記事は記事タイトルと抜粋、または本文から自動生成されます。
- 商品説明が短すぎる場合、ディスクリプションも短くなります。

## 13. 旧URLを維持するための注意点

このサイトはリニューアル後も旧本番URLを維持する前提です。

重要な旧URL例:

| 旧URL | 新サイトでの扱い |
| --- | --- |
| `/detail.html?detail_id=...` | 商品詳細 |
| `/detail_outlet.html?detail_id=...` | 売約済み商品詳細 |
| `/list.html?list_id=...` | 商品カテゴリー一覧 |
| `/search.html?query=...` | 商品検索 |
| `/newsXXXX.html` | ニュース詳細 |
| `/topic1.html` 〜 `/topic9.html` | ニュースカテゴリ一覧 |
| `/rental.html` | 美容機器レンタル系ページ |
| `/outlet.html` | 美容機器売却系ページ |
| `/contact/contact_*.html` | お問い合わせ |
| `/lookingfor.html` | サロンお問い合わせ |
| `/lookingforrental.html` | サロンお問い合わせ |

注意点:

- 固定ページのスラッグだけを変えても、旧URLルーティングは変わりません。
- 旧URLの記事や商品内容が違う場合は、投稿IDではなく `topic_id`、`custom_permalink`、`connect_id` を確認してください。
- 旧URLの移管対応はSEOに直結するため、削除やURL変更は事前確認が必要です。

## 14. 画像を差し替える

操作手順:

1. 管理画面の「メディア」を開きます。
2. 画像をアップロードします。
3. 投稿や固定ページのアイキャッチ、本文画像として設定します。

注意点:

- テンプレートに直接書かれている画像は、管理画面から差し替えできない場合があります。
- 商品画像は商品詳細データやアイキャッチ画像と連動している場合があります。
- 画像名を変えると既存リンクが切れる可能性があります。

## 15. 公開前チェック

ページを更新したら以下を確認してください。

| 確認項目 | 内容 |
| --- | --- |
| 表示確認 | PC、スマホで崩れていないか |
| リンク確認 | 主要ボタン、ナビ、CTAが正しいURLか |
| フォーム確認 | 送信できるか、メールが届くか |
| 商品確認 | 商品名、価格、画像、問い合わせ導線が正しいか |
| SEO確認 | title、descriptionが空でないか |
| 旧URL確認 | 旧URLでアクセスして200表示されるか |

## 16. よくあるトラブル

### 商品詳細が別の商品になる

確認する項目:

| 項目 | 確認内容 |
| --- | --- |
| `detail_id` | URLのIDが正しいか |
| `connect_id` | 投稿と商品データが正しく紐づいているか |
| カテゴリー | 商品カテゴリーに入っているか |

### ニュース詳細が別の記事になる

確認する項目:

| 項目 | 確認内容 |
| --- | --- |
| `topic_id` | 旧URLの数字IDと一致しているか |
| `custom_permalink` | `newsXXXX.html` が入っているか |
| 投稿ステータス | 公開済みになっているか |

### 商品一覧に出ない

確認する項目:

| 項目 | 確認内容 |
| --- | --- |
| 投稿ステータス | 公開済みか |
| カテゴリー | 商品カテゴリー配下に入っているか |
| 価格データ | 表示対象の価格があるか |
| 絞り込み条件 | 条件により非表示になっていないか |

### 問い合わせメールが届かない

確認する項目:

| 項目 | 確認内容 |
| --- | --- |
| 通常フォーム | Contact Form 7 のメール設定 |
| 商品問い合わせ | 設定 > 商品問い合わせメール |
| 送信先 | `order@respect-force.co.jp` になっているか |
| 迷惑メール | 受信側で迷惑メールになっていないか |

## 17. 運用上の注意

- URLは旧本番から引き継ぐ前提のため、公開後に安易に変更しないでください。
- 固定ページのテンプレート変更は表示崩れの原因になります。
- 商品やニュースの旧IDはSEO移管と表示内容に関わるため、削除しないでください。
- 大きな変更を行う前は、対象ページURLと変更内容をメモしておくと復旧しやすくなります。
- 不明な場合は、変更前に現在の画面や設定値をスクリーンショットで残してください。
